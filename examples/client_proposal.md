## 一、项目背景与目标

### 1.1 背景

随着企业数字化转型加速，内部积累了大量的非结构化文档数据（政策文件、合同文书、技术文档、培训材料等）。如何高效利用这些知识资产，成为企业提升效率的关键问题。

### 1.2 目标

建设企业级**智能知识库问答系统**，实现：

- ✅ 文档自动解析与向量化存储
- ✅ 基于自然语言的智能问答
- ✅ 回答可溯源，答案可解释
- ✅ 支持私有化部署，数据不出网络

## 二、技术方案

### 2.1 核心技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                      用户交互层                             │
│                  (Web界面 / 企业IM)                          │
└─────────────────────────┬───────────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────────┐
│                    RAG 查询流水线                            │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌─────────┐ │
│  │ Query    │ → │ Retrieval│ → │  Rerank  │ → │   LLM   │ │
│  │ 理解     │   │  检索    │   │  重排    │   │  生成   │ │
│  └──────────┘   └──────────┘   └──────────┘   └─────────┘ │
└─────────────────────────────────────────────────────────────┘
                          ▲
┌─────────────────────────▼───────────────────────────────────┐
│                    RAG 索引流水线                            │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌─────────┐ │
│  │ 文档解析 │ → │  文本    │ → │ Embedding│ → │  向量   │ │
│  │ (PDF等)  │   │  抽取   │   │  向量化  │   │  存储   │ │
│  └──────────┘   └──────────┘   └──────────┘   └─────────┘ │
└─────────────────────────────────────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────────┐
│                    基础设施层                                │
│        (向量数据库 / LLM API / 企业文件系统)                  │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 技术选型

| 组件 | 方案 | 说明 |
|---|---|---|
| RAG框架 | **RAGify**（自研） | 模块化设计，支持灵活扩展 |
| LLM | GPT-4o / Claude-3.5 | 支持客户指定模型 |
| Embedding | text-embedding-3 / BGE | 支持本地或云端 |
| 向量数据库 | Chroma / Milvus / Qdrant | 按规模选择 |
| 文档解析 | PDFMiner + Docling | 多格式支持 |
| 部署方式 | 私有化Docker部署 | 数据不出网络 |

### 2.3 RAGify 框架优势

相比开源项目（如 LangChain Demo），RAGify 具有以下差异化优势：

- **多模态原生支持**：图片、PDF、Word 统一处理
- **MCP流水线设计**：索引和查询流水线化，便于调试优化
- **内置Agent能力**：开箱即用的多类型Agent
- **YAML配置化**：无需改代码，配置即可切换组件
- **MIT协议可商用**：无商业授权纠纷
